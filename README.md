# Проявка — лендинг курса плёночной фотографии

**Демонстрационный проект.** Курс «Проявка», автор, работы выпускников и отзывы
вымышлены. Лендинг сделан как портфолио-работа.

> Статус: каркас репозитория. Разработка не начата.
> План реализации — [`docs/plan.md`](./docs/plan.md).

## О проекте

Лендинг офлайн-курса плёночной фотографии: съёмка, ручная проявка, печать. Цель —
бронь места в потоке до закрытия набора.

Тема выбрана намеренно нишевой и не-айтишной. Курс офлайновый по своей природе:
нужна лаборатория, поэтому ограниченное число мест — физический факт, а не
маркетинговая уловка. Таймер и «осталось 3 места» здесь честны.

Визуальная идея — **тёмная комната и контрольный лист**. Страница начинается в
темноте под красным светом фонаря и «проявляется» по мере скролла: нижние секции
уже бумага, тёплая и светлая. Метафора процесса, положенная на скролл, двумя
цветами фона и без единой строки JS.

## Что здесь показано технически

Проект про **состояние и время** — три вещи, которые почти везде делают неправильно:

- **Таймер не верит клиенту.** `Date.now()` в браузере — это системное время
  пользователя, которое он может перевести. Смещение до серверного времени
  считается один раз при монтировании.
- **Таймер не ломает гидратацию и не двигает макет.** Живое значение появляется
  после монтирования, на месте цифр — прочерки той же ширины (моноширинный шрифт).
  CLS от таймера равен нулю.
- **Таймер не отстаёт в фоновой вкладке.** Каждый тик считается разница с дедлайном,
  а не декремент счётчика.
- **Условная логика потоков** — одна чистая функция `resolveCohortState()`, покрытая
  тестами, из которой выводятся заголовок, таймер и текст кнопки. Когда все потоки в
  конфиге истекли, страница показывает лист ожидания, а не отрицательный отсчёт.
- **Zustand используется по делу**: в сторе только выбранный тариф и состояние
  модалки. Тиков таймера там нет — они бы перерисовывали дерево раз в секунду.

## Как начать

Репозиторий пока содержит только документацию и конфигурацию. Скаффолд:

```bash
pnpm create next-app@latest . --typescript --tailwind --app --eslint --src-dir
cp .env.example .env.local   # и заполнить
pnpm dev
```

Порядок работы: сначала `resolveCohortState()` с юнит-тестами, потом вёрстка.
Это код, где ошибка не видна глазами — она проявится через месяц на проде.
